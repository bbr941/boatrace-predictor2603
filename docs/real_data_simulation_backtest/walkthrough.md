# 完了報告: 実データ・シミュレーションバックテスト

`boatrace.db` の実データを用いたバックテスト環境を構築し、シミュレーション・パラメータ（各特徴量の重み）の最適化と検証を行いました。

## 実施内容

### 1. 実データ接続と抽出
- [monte_carlo_backtest.py](file:///d:/BOAT2512_AntiGravity_2_ana/monte_carlo_backtest.py) を実装し、大規模な `boatrace.db` から直近100レース分の特徴量と結果を高速に抽出するパイプラインを構築しました。

### 2. 特徴量の正規化と時系列分割
- `MinMaxScaler` を用いて、勝率や展示タイムなどのスケールの異なる変数を統合。
- データを日付順にソートし、7:3 の比率で Training Set（学習）と Test Set（評価）に分割しました。

### 3. Optunaによる重み最適化とバックテスト
- **学習フェーズ**: Training Set に対して、的中頻度を維持しながらROIを最大化するように重みを探索。
- **評価フェーズ**: 学習で得られた「最良の重み」を Test Set（未知のデータ）に適用し、パフォーマンスを測定しました。

## バックテスト結果（例）

100レース（Train 70 / Test 30）での実行結果は以下の通りです：

- **学習期（Train）**: ROI 183.7% （最適化が成功し、高い収益性を記録）
- **検証期（Test）**: ROI 0.0% （未知の30レースでは的中なし）
- **ベット機会**: 36回（1レース平均 1.2点）

> [!NOTE]
> テスト期間でのROIが振るわなかった原因として、サンプルサイズ（30レース）の少なさや、シミュレーションモデルの極めてシンプルな性質（高度な物理挙動の欠如）が考えられます。しかし、**「学習と評価を分離してモデルの脆さを可視化する」**というバックテスト本来の目的は達成されました。
