# 実装計画: モンテカルロ・シミュレーション プロトタイプ

競艇（ボートレース）の3連単出現確率を、スタートタイミングのシミュレーションとコース別勝率を組み合わせて予測するプログラムのプロトタイプを作成します。

## 提案する変更

### [Component Name] [Monte Carlo Simulation Engine]

#### [NEW] [monte_carlo_simulation.py](file:///d:/BOAT2512_AntiGravity_2_ana/monte_carlo_simulation.py)
シミュレーションのメインスクリプトです。以下のロジックを含みます。

1.  **ダミーデータ定義**: 
    - 6艇分の「平均ST」「ST標準偏差」「コース別1-3着勝率」を `pd.DataFrame` で保持します。
2.  **スタートタイミング生成**: 
    - `np.random.normal` を使用し、10,000回分の各艇のスタートタイミング（ST）をシミュレートします。
3.  **展開ロジック**: 
    - スリット隊形（STの差）に基づいて、有利・不利の係数を算出します。
    - 簡易的な「捲り（まくり）」判定：隣接する艇より一定（例: 0.1秒）以上早い場合に、その艇とさらに内側の艇の勝率を調整します。
    - 各スロットの期待順位を算出します。
4.  **期待値計算とフィルタリング**:
    - 全120通りの3連単組み合わせを生成。
    - 仮想オッズ（正規分布等で生成したダミー値）を用いて期待値を計算。
    - 引数 `threshold` によって買い目を出力します。

## ユーザーレビューが必要な項目

> [!IMPORTANT]
> 展開ロジックは今回「簡易版」として実装します。特に「1号艇が凹んだ場合」や「4号艇のカド捲り」などを簡単な条件分岐で表現しますが、より高度な物理挙動は将来の拡張課題とします。

## 検証計画

### 自動テスト
- `monte_carlo_simulation.py` を直接実行し、出力される期待値と買い目リストの整合性を確認します。
- 3連単の出現確率の総和が1.0に近似することを確認します。

### 手動検証
- 期待値閾値（1.05, 1.1など）を変更した際に、ヒットする買い目の数（ベット機会）が適切に増減することを確認します。
