# 実装計画: プランB（購入率50%制限フィルター）の発動と自動最適化

## 目的
プランAの資金傾斜配分ベースラインに対し、モデルの自信度を表す指標（軸艇の信頼度、上位の明瞭度）に基づくレースフィルターを導入します。また、シミュレーション処理を構造化し、参戦率約50%を維持しつつ最高ROIを叩き出す閾値を自動探索（グリッドサーチ）する機能を実装します。

## 提案する変更

### `simulate_betting.py` の構造リファクタリング（高速グリッドサーチ対応）
毎回のパラメータ変更で重いモデル推論やオッズ取得を繰り返さないよう、処理フェーズを2段階に分離します。

1. **フェーズ1（予測・配分計算のストック）**:
   これまで通り全レースに対して、推論、PL確率計算、Plan Aの資金傾斜配分による購入目と金額（および的中時のリターン）を算出し、その結果をリストに一時保存します。
   また同時に、各レースのフィルター指標を計算し保存します。
   - `max_p1`: 1着確率（Softmax）の最大値（＝軸艇の信頼度）
   - `prob_gap`: PL合成確率の1位と2位の差（＝上位の明瞭度）

2. **フェーズ2（グリッドサーチと最終集計）**:
   - `USE_PLAN_B = True` の場合、ストックされた全レース結果に対して閾値探索を行います。
   - `max_p1` の閾値（例: 0.15〜0.50）と `prob_gap` の閾値（例: 0.00〜0.10）のあらゆる組み合わせをループし、購入対象となるレースをフィルタリングします。
   - 「購入率（参戦レース数 / 全レース数）が 45% 〜 55% に収まる」という条件を満たすパラメータ群の中で、**最も回収率（ROI）が高くなる組み合わせ**を最適解として特定します。

3. **出力レポートの拡張**:
   最適化された閾値（採用パラメータ）を表示し、プランA（フィルタなし）とプランB（フィルタあり）の成績を比較できるサマリーログを出力します。
