# Streamlit アプリケーションへの「プランB」統合完了報告

ご指示いただいた通り、マルチフォーメーション検証でその強力さが証明された**「プランB（純粋なPlackett-Luce確率抽出 ＋ 穴モデルハイブリッド ＋ 資金傾斜配分）」**のロジックを、Streamlitアプリ（`app_boatrace.py`）に完全に統合しました。

## 実装内容と変更点

1. **デュアルモデルへの完全移行**
   これまでの本命モデル単独での運用を廃止し、`model_honmei.txt` と `model_ana.txt` の両方をロードして予測を行う仕様に変更しました。これにより、アプリ上で穴展開の可能性も同時に評価できるようになりました。

2. **Plackett-Luce確率算出エンジンの搭載**
   アプリ内で推論した本命スコアを即座にSoftmax関数にかけ、全120通りの3連単の正確な合成確率を算出するロジック（`calculate_plackett_luce_probs`）を移植しました。

3. **旧UIの刷新と勝負レースフィルター**
   標準偏差に基づく古い「Enjoy / Chaos / Skip」の判定を完全に撤廃しました。
   代わりに、グリッドサーチで導き出された最も信頼性の高いフィルター（`max_p1 >= 0.49` かつ `prob_gap >= 0.010`）を実装しました。
   条件を満たしたレースは**「Target Race (Plan B)」**として強く推奨し、満たさない場合は「ケン（Skip）」として警告するUIに刷新しています。

4. **予算設定（サイドバー）と資金傾斜配分**
   サイドバーに以下の予算設定項目を追加しました。
   - **Base Budget (Flat Payout):** デフォルト1000円
   - **Bonus Budget (EV Boost):** デフォルト500円
   これらの予算とPlackett-Luce確率を元に、買い目ごとの最適な購入金額（トリガミ排除済み）を動的に計算する資金配分ロジック（`calculate_funds_distribution`）を搭載しています。

5. **シミュレーションコードのロールバック**
   `simulate_betting.py` から成績が悪化したマルチフォーメーション型のコードを削除し、純粋なプランB（トップダウン抽出ロジック）でグリッドサーチ最適解（P1≧0.49 等）のハードコーディングを含む形にロールバックして保存しました。

> [!NOTE]
> **オッズ取得に関する仕様について**
> 資金配分機能の動作（UI）を確認していただくため、現状はランダムシードを用いたダミーの3連単オッズ（15.0〜50.0倍程度）を生成して買い目ごとの金額を計算し、テーブルに表示しています。実際の舟券購入に直結させる場合は、3連単オッズのリアルタイムスクレイピング機能の追加実装が必要となります。

アプリを起動して、サイドバーでの予算変更や、ターゲットレースでの買い目・投資金額の算出UIをご確認ください！
