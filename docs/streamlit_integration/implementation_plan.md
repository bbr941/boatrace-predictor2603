# 実装計画: Streamlitアプリへの「プランB」ロジックの統合

## 目的
マルチフォーメーション検証でその数学的優位性が完全に証明された**「プランB（純粋なPlackett-Luce上位抽出 ＋ 穴ハイブリッド ＋ 資金傾斜配分）」**のロジックを、実際の運用ツールであるStreamlitアプリ（`app_boatrace.py`）に統合します。
これにより、これまでの「Enjoy / Chaos / Skip」といった旧基準を廃止し、最新のAI予想エンジンへとアップデートします。

## 提案する変更（`app_boatrace.py`）

1. **デュアルモデル推論の統合**:
   - `model_honmei` と `model_ana` の両方をロードし、それぞれのスコアを推論します。

2. **Plackett-Luce確率計算の導入**:
   - `simulate_betting.py` で実績のある `calculate_plackett_luce_probs` 関数をアプリ内に移植し、全120通りの正確な合成確率を算出します。

3. **プランB（買い目抽出・資金配分）ロジックの実装**:
   - リアルタイムで取得したオッズ（`get_odds`）に基づき、動的購入上限（$N = \min(8, \lfloor Odds \rfloor)$）を決定します。
   - PL確率の上位 $N$ 点を抽出し、穴モデルのトップ艇を用いたハイブリッド入れ替え処理を行います。
   - `calculate_funds_distribution` 関数を移植し、期待値ブースト型の資金傾斜配分（ベース1000円＋ボーナス500円）を計算し、トリガミを完全排除します。

4. **プランB 勝負レースフィルターの適用**:
   - 先のグリッドサーチで導き出された最適閾値（`max_p1 >= 0.49` かつ `prob_gap >= 0.010`）をフィルターとして設定します。
   - 閾値を満たさないレースは「ケン（見送り推奨）」として表示し、満たした場合は資金配分付きの買い目リストを大々的に表示します。

5. **`simulate_betting.py` のロールバック**:
   - マルチフォーメーション型のコードを破棄し、純粋なプランB（トップダウン抽出）のコードにロールバックして保存しておきます（今後の検証用）。
