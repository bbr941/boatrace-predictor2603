# 実装計画: 本命・穴共存型「2軸変則フォーメーション」

## 目的
1レースの買い目（最大8点）の中に「本命の軸」と「穴の刺客」を強制的に共存させる2軸フォーメーションを実装し、的中率を維持しつつ、荒れたレースでの高配当を拾い上げて回収率（ROI）の向上を狙います。

## 提案する変更

### `simulate_betting.py` の買い目生成ロジック改修
`select_hybrid_formation` をベースに、以下の新しい「共存型フォーメーション」ロジックに置き換えます。

#### ステップ1: 構成メンバーの選定
1. **1着候補（軸・1艇）**: 本命モデルの1着確率（Softmax値）が最も高い艇。
2. **2着候補（本命＆穴・計2艇）**:
   - **本命対抗（1艇）**: 1着候補を除いた中で、本命確率が最も高い艇。
   - **穴の刺客（1艇）**: 穴モデル（`score_ana`）が最も高い艇。※1着候補や本命対抗と重複した場合は、穴スコア2番手以降の艇へとスライドさせ、必ず「異なる2艇」を確保します。
3. **3着候補の展開**: 1着・2着の組み合わせに対して、残りの艇を3着として展開します。

#### ステップ2: 8点の展開と動的点数制限（N）
1. 選定されたメンバーから、ベースとなる「8点」の3連単の組み合わせを生成します。
2. 生成された8点の各買い目に対して、Plackett-Luceモデルで計算された合成確率（PL確率）を付与し、確率が高い順に並べ替えます。
3. 確率1位のオッズから上限 $N = \min(8, \lfloor Odds_{\text{top}1} \rfloor)$ を計算し、上位から $N$ 点を抽出します（ガチガチのレースでは、確率が低い穴絡みの目がここで自動的に削ぎ落とされます）。

#### ステップ3: 資金傾斜配分（プランA）の実行
- 厳選された $N$ 点に対し、前回実装の `calculate_funds_distribution` を通して資金配分とトリガミ排除を行います。
