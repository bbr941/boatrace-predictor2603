# 実装計画: 3連単フォーメーションベッティング及び動的点数制限の実装

## 目的
本命モデルの予測スコアをSoftmax+Plackett-Luceモデルで確率に変換し、リアルタイムオッズに基づく動的な点数制限（最大8点）と、穴モデルを組み合わせたハイブリッドフォーメーション戦略を実装します。さらにトリガミを完全に排除する最終オッズフィルターを導入します。

## 提案する変更

### `simulate_betting.py` の改修

現在の「生のスコアによる上位抽出＋資金配分（階段買い）」ロジックを完全に置き換えます。

#### 1. 確率計算関数の新規追加
- `calculate_plackett_luce_probs(honmei_scores)`: 
  本命モデルの6艇のスコア（生値）をSoftmaxで1着確率に変換し、Plackett-Luceモデルを用いて3連単全120通りの合成確率を計算してリストとして返す関数を実装します。

#### 2. 全オッズ一括取得ロジックの追加
- データベースから特定の買い目だけでなく、そのレースの3連単の全オッズ（120通り）を一括で取得する関数 `get_all_trifecta_odds(conn, race_id)` を追加し、計算効率を高めます。

#### 3. 動的フォーメーション＆ハイブリッド抽出関数の実装
- `select_hybrid_formation(pl_probs, ana_scores, all_odds)`:
  1. PL確率1位の買い目のオッズを確認し、上限点数 $N = \min(8, \lfloor Odds_{\text{top}1} \rfloor)$ を決定（$N<1$なら見送り）。
  2. 確率上位 $N$ 点を抽出。
  3. 穴モデルのスコアが最も高い艇（$B_{ana}$）を特定。
  4. 抽出した $N$ 点の中に「3着に $B_{ana}$」が含まれていない場合、PL確率1位の買い目の「1着・2着」と「$B_{ana}$」を組み合わせた新しい買い目を生成し、確率最下位の買い目と入れ替える。

#### 4. トリガミ排除とフラット買い（一律購入）の統合
- 選定された最大 $N$ 点のオッズをチェックし、「**すべての買い目のオッズが $N$ 以上であること**」を確認します。条件を満たさない場合、そのレースの購入をすべて見送る（ケン）処理とします。
- 資金配分は `staircase_betting_logic` を廃止し、すべて1点100円のフラット買いとしてシミュレーションの統計を計算します。

#### 5. 評価ログの拡張
- 回収率、的中率、購入レース数（購入率）の変化を確認できるよう、サマリーの出力を調整します。

> 1. **穴モデルによる追加買い目の「1着・2着」の決め方について**

> 
> 2. **トリガミ排除時の挙動について**
>    $N$点の中にオッズが $N$ 未満の買い目が1つでも含まれていた場合、「その目だけを削除して購入する」か「レース全体の購入を見送る（ケン）」の2つのアプローチがあります。フォーメーションの意図を崩さず堅実に運用するため、今回は**「レース全体を見送る（ケン）」**とする方針で進めてよろしいでしょうか？
