# 不整合修正 & スクレイピング透明化 (V3.1+) 完了報告書

## 実施した修正項目

### 1. 計算結果の完全な再現性（決定論的システム）
- **モデルのキャッシュ化**: 
    - `@st.cache_resource` を `load_models` に適用しました。これにより、アプリ操作中にモデルが再ロードされることがなくなり、推論に使用される重みが完全に固定されます。
- **ランダム性の完全排除**: 
    - 以前のコードに含まれていた `np.random.dirichlet`（確率生成）および `np.random.lognormal`（オッズ生成）のモックロジックを**全て削除**しました。
    - 的中確率は、実証された stacking モデルの構成（LGBM + CatBoost + Alt-LR）による計算のみで導出されます。
    - 同じレース、同じ設定（予算・閾値）に対しては、何度計算ボタンを押しても 1円単位で同じ結果が返ることを保証しました。

### 2. スクレイピングの正確性と透明化（デバッグモード）
- **オッズ不一致の解消**: 
    - **原因特定**: 以前のバージョンでは、UIのデモを優先するためにオッズをランダムな数値でシミュレート（モック）しており、公式サイトの値と物理的に不一致が発生していました。
    - **対策**: `data_fetcher.py` を完全に統合し、公式サイトから取得した「生オッズ」を直接ベッティング戦略に使用するように修正しました。
- **デバッグ機能の実装**:
    - サイドバーに **[Debug Mode]** トグルを追加しました。
    - オンにすると、以下の情報をエクスパンダー内で確認できます。
        - **取得日時とURL**: どのレースに対していつアクセスしたか。
        - **推論特徴量 (X)**: AI が予測の根拠とした数値（展示タイム、勝率、モーター率など）の表。
        - **生 JSON 抽出結果**: 展示進入コースやスタートタイミングの抽出内容。
        - **生オッズサンプル**: サイトから読み取った直後のオッズ値。

## 今後の運用
これで「なぜその買い目になったのか」をユーザー自身で生データを見て納得できる環境が整いました。不一致が発生した場合は、デバッグビューの「Target URL」を直接ブラウザで開き、サイト上の数値と突き合わせることが可能です。

streamlit run boatrace-v3-predictor/app_v3.py
