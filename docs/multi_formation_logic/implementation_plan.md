# 実装計画: マルチフォーメーション自動選択ロジック

## 目的
レースの確率分布（1着確率の圧倒度合い、2強の競り合い、上位3艇の占有率）に応じて、「本命型」「裏表型」「ボックス型」の3つのフォーメーションを自動的に切り替え、荒れるレースでの高配当獲得と参戦率の最適化（約50%でROI最大化）を両立します。

## 提案する変更

### `simulate_betting.py` の構造リファクタリング

#### フェーズ1: テンプレート買い目の事前生成
各レースにおいて、以下の3パターンのベース買い目をすべて事前生成し、キャッシュに保存します。

1. **【本命型】1軸・強め流し（計8点）**
   - 1着: P1確率1位
   - 2着: P1確率2位 ＋ 穴スコア1位（重複時はスライド）
   - 3着: 残り4艇すべて
2. **【裏表型】2軸・クロス（※要確認: 計4点 or 8点）**
   - 1着・2着: P1確率1位・2位のクロス
   - 3着: P1確率3位 ＋ 穴スコア1位（重複時はスライド）
3. **【ボックス型】3連単BOX（計6点）**
   - 構成艇: P1確率1位、2位 ＋ 穴スコア1位（重複時はスライド）

#### フェーズ2: 動的グリッドサーチ（3次元）
キャッシュされた全レースに対して、以下の優先順位と閾値で適用するフォーメーションを決定するシミュレーションを回し、最適パラメータを探索します。

1. **BOX判定**: 上位3艇の1着確率の合計 $\ge$ `th_box` なら【ボックス型】を適用
2. **裏表判定**: 上位2艇の1着確率の差 $\le$ `th_gap` なら【裏表型】を適用
3. **本命判定**: 1位の1着確率 $\ge$ `th_honmei` なら【本命型】を適用
4. 該当なし: ケン（見送り）

選択されたフォーメーションに対して、PL確率によるソート、オッズに基づく動的 $N$ 点抽出、および資金傾斜配分（トリガミ排除）を適用して成績を集計します。

> [!IMPORTANT]
> ## User Review Required (要確認事項)
> 
> 1. **【裏表型】の買い目点数（数学的確認）**
>    「1着・2着をAとBの裏表クロス、3着をCとDの固定2艇」とした場合、展開される買い目は `A-B-C`, `A-B-D`, `B-A-C`, `B-A-D` の **「計4点」** となります。ご指示には「計8点」とありましたが、**今回はこの4点のままとするか、あるいは3着候補を残り4艇すべてにして8点とするか**、どちらがご希望でしょうか？
> 
> 2. **テンプレートの優先順位について**
>    1つのレースが複数の条件（例：1位が確率0.45以上だが、上位3艇の合計も0.80以上ある）を満たす場合があります。その際、**「1. ボックス型 ＞ 2. 裏表型 ＞ 3. 本命型」** の順番で条件判定（どれか1つに決める）を行おうと考えていますが、この優先順位でよろしいでしょうか？
> 
> 3. **グリッドサーチの探索範囲について**
>    以下の範囲で自動探索をかけます。問題なければこのまま進めます。
>    - `th_box` (上位3艇の和): 0.60 〜 0.85 (0.05刻み)
>    - `th_gap` (1位と2位の差): 0.01 〜 0.06 (0.01刻み)
>    - `th_honmei` (1位の確率): 0.35 〜 0.50 (0.05刻み)
