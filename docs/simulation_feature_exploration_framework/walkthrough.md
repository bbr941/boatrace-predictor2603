# 完了報告: シミュレーション特徴量探索フレームワーク

シミュレーションの展開ロジックにおける各要素の重要度を自動探索するフレームワークを実装・検証しました。

## 実施内容

### 1. 多角的な実験環境の構築
- `monte_carlo_experiment.py` を作成し、全国勝率、當地勝率、モーター、展示タイム、ST、風速などの特徴量を網羅したダミーレース環境（30レース分）を構築しました。

### 2. 重み付きシミュレーションエンジンの実装
- 各特徴量に対して「重み（weight）」を動的に適用し、その合計スコアによって勝率を算出するロジックを実装しました。
- 重みが0に近づいた特徴量は無視されるため、自動的な「変数選択」が可能です。

### 3. OptunaによるROI最適化
- 回収率（ROI）を最大化するように重みを自動探索します。
- **的中頻度制約**: 購入レース数が一定以下（20%以下）になるとスコアに大きなマイナスペナルティを与えるロジックを組み込み、「回収率は高いが全く買わない」という極端な最適化を防止しました。

## 実験結果

30回の試行（Trials）の結果、以下のような重み（重要度）が算出されました：

- **High Importance**: `national_win_rate` (4.8093), `wind_speed` (4.3903)
- **Medium Importance**: `avg_st` (2.1333)
- **Low/Noise**: `local_win_rate`, `motor_rate`, `exhibition_time`

この結果から、今回のシミュレーション環境においては「全国勝率」と「風」がROI向上に最も寄与しており、逆に「モーター2連対率」などはノイズとして扱われていることが客観的に示されました。
