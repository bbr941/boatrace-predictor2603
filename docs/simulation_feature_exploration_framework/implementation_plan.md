# 実装計画: シミュレーション特徴量探索フレームワーク

シミュレーションの展開ロジックにおける各要素（特徴量）の寄与度を自動的に探索・評価する実験環境を構築します。

## 提案する変更

### [Component Name] [Experiment Framework]

#### [NEW] [monte_carlo_experiment.py](file:///d:/BOAT2512_AntiGravity_2_ana/monte_carlo_experiment.py)
実験用メインスクリプト。以下のモジュール構成とします。

1.  **DataLoader (Dummy Generator)**:
    - 6艇分の特徴量（全国・当地勝率、モーター2連対率、展示タイム等）に加えて、外部環境（風速・風向）を持つ複数レース分のダミーデータを生成。
2.  **Weighted Simulation Engine**:
    - 各特徴量に対する `weight` を受け取ります。
    - `score = sum(feature_i * weight_i) + noise` 形式で艇のポテンシャルを算出し、モンテカルロ法で勝率を導出します。
3.  **Objective Function (Optuna)**:
    - `params` として各重みを設定（0.0 〜 1.0）。
    - 各レースで期待値 > 閾値 の買い目を「擬似ベット」し、合計回収率を計算。
    - **Constraint**: 購入レース数が全体のN%（例: 20%）を下回る場合、ROIに大きなマイナスペナルティを与えて探索範囲を調整。
4.  **Result Analysis**:
    - 探索後の最適パラメータ（重み）を表示。
    - 重みの大きさを「重要度」として解釈し、不要なノイズ変数を特定。

## ユーザーレビューが必要な項目

> [!CAUTION]
> 今回は「ダミーデータ」を用いたフレームワーク構築が主目的ですが、将来的に `boatrace.db` と接続する際のデータのインターフェースについても考慮して設計します。

## 検証計画

### 自動テスト
- `monte_carlo_experiment.py` を実行し、Optunaの試行が正常に完了することを確認。
- 重みを意図的に1つだけ大きくした場合に、その特徴量が結果に反映されるかを確認。

### 手動検証
- 探索結果ログに出力される「重要度」が、直感的なボートレースの定石（例：1号艇の勝率が強いなど）と整合するか確認。
- ベット頻度ペナルティの有無で、出力される買い目数に差が出るかを確認。
